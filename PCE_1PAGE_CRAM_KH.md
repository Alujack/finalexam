# PCE — សន្លឹកសង្ខេប ១ ទំព័រ (ភាសាខ្មែរ)

## គន្លឹះធំ
សំណួរ ១២ កើតចេញពី **ប្លុក** ពីរបី ដែលប្រើឡើងវិញ។ រៀនប្លុក (អក្សរកាត់) ជាមុនសិន រួចរៀន **ពាក្យគន្លឹះ** សម្រាប់សំណួរនីមួយៗ។ ពេលប្រឡង ពន្លាតពាក្យគន្លឹះជាប្រយោគ។ ពិន្ទុមកពី **ចំណុច** មិនមែនពាក្យពិតប្រាកដ។

> អក្សរកាត់ (FATP, CIA ។ល។) រក្សាជាភាសាអង់គ្លេស ព្រោះអ្នកសរសេរវាក្នុងវិញ្ញាសាអង់គ្លេស។ សៀវភៅយោង = **Laudon**។

---

## ផ្នែកទី ១ — ប្លុកសំខាន់ (រៀនមុនគេ — អានឱ្យឮៗ)

- **FATP** = គោលការណ៍សីលធម៌វិទ្យាសាស្ត្រទិន្នន័យ ៤ → **F**airness (យុត្តិធម៌) · **A**ccountability (គណនេយ្យភាព) · **T**ransparency (តម្លាភាព) · **P**rivacy (ឯកជនភាព)
- **CIA** = CIA Triad (សុវត្ថិភាព) → **C**onfidentiality (សម្ងាត់) · **I**ntegrity (ត្រឹមត្រូវ) · **A**vailability (អាចប្រើបាន)
- **DCVC** = ធាតុ Informed Consent ៤ → **D**isclosure (បង្ហាញ) · **C**omprehension (យល់) · **V**oluntariness (ស្ម័គ្រចិត្ត) · **C**ompetence (សមត្ថភាព)
- **AER** = សិទ្ធិ GDPR ៣ → **A**ccess (ចូលប្រើ) · **E**rasure (លុប/right to be forgotten) · **R**ectification (កែតម្រូវ)
- **MPD** = ការគំរាមកំហែងសុវត្ថិភាព ៣ → **M**alware (Virus/Worm/Trojan) · **P**hishing/spoofing · **D**oS/DDoS
- **MEA** = ការអនុវត្តល្អ ៣ (privacy in software) → **M**inimisation (ប្រមូលតិច) · **E**ncryption (អ៊ិនគ្រីប) · **A**ccess control (គ្រប់គ្រងការចូល)
- **គោលការណ៍ Laudon** = Golden Rule · Categorical Imperative · Slippery slope · Utilitarian · Risk-aversion · (FIP = Fair Information Practices)
- **ឧទាហរណ៍ bias** = Amazon (recruiting, រើសអើងស្ត្រី) · COMPAS (តុលាការ, រើសអើងជនស្បែកខ្មៅ)

## ផ្នែកទី ២ — ពាក្យគន្លឹះ ១២ សំណួរ

1. សារៈសំខាន់ក្រមសីលធម៌ IT = ត្រូវ/ខុស; បច្ចេកវិទ្យាបង្កើនបញ្ហា (profiling); **ឧទាហរណ៍៖** Facebook–Cambridge Analytica, ការឃ្លាំមើលបុគ្គលិក, Equifax breach; **ផល៖** បុគ្គល=ការពារឯកជនភាព; ស្ថាប័ន=ទំនុកចិត្ត/ជៀសបណ្ដឹង
2. គោលការណ៍ ៤ វិទ្យាសាស្ត្រទិន្នន័យ = **FATP** (នីមួយៗ + ឧទាហរណ៍ ១)
3. Framework ធៀប Regulation = guideline ស្ម័គ្រចិត្ត (ACM Code, **should**) ធៀប regulation ច្បាប់ (HIPAA/COPPA/GDPR, **must**)
4. Algorithmic bias = លទ្ធផលរើសអើងជាប្រព័ន្ធ (ទិន្នន័យលម្អៀង); **ឧទាហរណ៍ ២៖** Amazon + COMPAS
5. ហេតុអ្វី fairness សំខាន់ក្នុង AI = AI សម្រេចរឿងធំ (ការងារ/កម្ចី/ប៉ូលីស); បើមិនយុត្តិធម៌→គ្រោះថ្នាក់, បង្កើនវិសមភាព, បាត់ទំនុកចិត្ត, ទទួលខុសត្រូវផ្លូវច្បាប់; Golden Rule
6. GDPR = ច្បាប់ការពារទិន្នន័យ EU (2018); **សិទ្ធិ ៣ = AER** (Access, Erasure, Rectification)
7. Informed consent = យល់ព្រមដោយដឹងការពិត មុនប្រមូល; **ធាតុ ៤ = DCVC**; ឧទាហរណ៍៖ cookie opt-in, បែបបទយល់ព្រមវេជ្ជសាស្ត្រ
8. CIA Triad = **CIA** (Confidentiality=អ៊ិនគ្រីប; Integrity=hashing; Availability=backup; ការគំរាមកំហែង=DoS)
9. ការគំរាមកំហែងសុវត្ថិភាព = **MPD** (Malware: Virus/Worm/Trojan; Phishing/spoofing; DoS/DDoS)
10. ឯកជនភាពក្នុង software = សំខាន់ (ការលួចអត្តសញ្ញាណ/ច្បាប់); "privacy by design"; **ការអនុវត្ត ៣ = MEA**
11. Explainability = យល់ថា **ហេតុអ្វី** AI សម្រេច (មិនមែន black box); សំខាន់៖ គណនេយ្យភាព, រកភាពលម្អៀង, ទំនុកចិត្ត, GDPR; ឧ. កម្ចីបដិសេធ "ដោយសារចំណូលទាប"
12. អនុវត្ត PCE អនាគត = ① privacy/security by design (MEA, CIA) ② សាកល្បង fairness/bias (Amazon) ③ ប្រើគោលការណ៍ Laudon + ACM Code ពេលប្រឈមបញ្ហា

---

## ផ្លូវកាត់៖ ចម្លើយប្រើឡើងវិញ
- **CIA / MEA / encryption** ប្រើនៅ Q8, Q10, Q12 — រៀនម្ដងឆ្លើយ ៣
- **FATP / fairness / bias** ប្រើនៅ Q2, Q4, Q5, Q12
- **Amazon + COMPAS** ប្រើបានទាំង Q4 និង Q5
→ ពិតប្រាកដមានរឿងត្រូវរៀនតែប្រហែល **៦-៧** មិនមែន ១២ ទេ។

## ផែនការសិក្សា & ច្បាប់
1. **ម៉ោង ១** — ទន្ទេញអក្សរកាត់ (FATP, CIA, DCVC, AER, MPD, MEA) ឱ្យឮៗ
2. **ម៉ោង ២** — អានសំណួរ → និយាយពាក្យគន្លឹះ → និយាយចម្លើយ → ពិនិត្យ
3. **ម៉ោង ៣** — សាកល្បងតែសំណួរដែលខុស + សរសេរ ៣ ចម្លើយលើក្រដាស
4. **ម៉ោង ៤** — អានម្ដងទៀត រួចគេង
- **និយាយឱ្យឮ កុំអានស្ងាត់** · **ពិន្ទុមកពីចំណុច** · សរសេរជា **bullet points** ពេលប្រឡង · ដាក់ឧទាហរណ៍ ១ (Amazon/COMPAS/Facebook)
